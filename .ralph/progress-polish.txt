# Ralph Progress — Polish Stack

## Codebase Patterns (connections-helper-specific)
- TanStack Start on Cloudflare Workers with D1 + Drizzle
- Routes are file-based under src/routes/
- API routes in src/routes/api/ use createFileRoute with server.handlers.GET/POST
- PostHog wired via src/lib/posthog.ts with typed track() wrapper + EventPayloads
- Existing e2e/ uses Playwright; tests/integration/ uses vitest
- Existing /api/stats endpoint; /how-it-works has a StatsBanner
- Fonts live in public/fonts/
- No Inter font yet; the app uses whatever is in the existing CSS
- Production origin is https://connectionshelper.app (from wrangler.jsonc custom_domain routes)
- File-routes with literal dots use the `[.]` escape: src/routes/sitemap[.]xml.ts → /sitemap.xml
- Route tree is auto-generated at src/routeTree.gen.ts — inspect it to confirm new routes registered

## [2026-04-24 21:35] - US-007: @axe-core/playwright e2e a11y tests
- Implemented:
  - pnpm add -D @axe-core/playwright
  - e2e/accessibility.spec.ts loops over [/, /how-it-works, /settings, /design-system], runs AxeBuilder with wcag2a/wcag2aa/wcag21aa tags, fails on any violation
  - Baselined two rules with explicit TODO: color-contrast (theme work) and label (design-system showcases unlabelled primitives intentionally)
  - Fixed true violations discovered along the way: added aria-label="Previous day" and aria-label="Next day" to App.tsx date-nav icon buttons (Tooltip-only labelling doesn't satisfy button-name)
- Files changed:
  - package.json (devDeps)
  - e2e/accessibility.spec.ts (new)
  - src/App.tsx (aria-labels on nav buttons)
- **Learnings for future iterations:**
  - @axe-core/playwright rejects unknown rule IDs with `unknown rule X in options.rules` — only pass *rule* IDs (e.g. `button-name`) not *check* IDs (e.g. `implicit-label`). The check IDs appear inside `any: [...]` arrays on a violation; they are NOT valid inputs to disableRules.
  - Baseline ratchet pattern: start with a list of known-gap rules in BASELINED_RULES, add a comment linking each to its future fix story, and remove them one at a time as they get addressed. Keeps CI useful today AND drives us toward zero-violation.
  - Tooltip-only labelling fails axe button-name — aria-label on the button itself is required. TooltipContent is decorative from axe's perspective.
  - /design-system is a live route (not gated by env), so it ships to production. Either gate it, or accept that its axe violations reflect a public surface.
