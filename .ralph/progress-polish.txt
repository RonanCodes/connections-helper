# Ralph Progress — Polish Stack

## Codebase Patterns (connections-helper-specific)
- TanStack Start on Cloudflare Workers with D1 + Drizzle
- Routes are file-based under src/routes/
- API routes in src/routes/api/ use createFileRoute with server.handlers.GET/POST
- PostHog wired via src/lib/posthog.ts with typed track() wrapper + EventPayloads
- Existing e2e/ uses Playwright; tests/integration/ uses vitest
- Existing /api/stats endpoint; /how-it-works has a StatsBanner
- Fonts live in public/fonts/
- No Inter font yet; the app uses whatever is in the existing CSS
- Production origin is https://connectionshelper.app (from wrangler.jsonc custom_domain routes)
- File-routes with literal dots use the `[.]` escape: src/routes/sitemap[.]xml.ts → /sitemap.xml
- Route tree is auto-generated at src/routeTree.gen.ts — inspect it to confirm new routes registered

- **Learnings for future iterations:**
  - CF workerd blocks `WebAssembly.instantiate(bytes)` (dynamic code gen) in both dev and prod. Default `satori` entry hits this path. Fix: use `satori/standalone` — it exposes `init(WebAssembly.Module)` and lets you feed a pre-compiled module (imported via vite). Same pattern applies to `@resvg/resvg-wasm`'s `initWasm()`.
  - Fonts: `?url` imports return a URL string; fetching that URL against `url.origin` works because vite dev and the CF asset pipeline both serve bundled `/src/assets/...` paths as the binary. An alternative `?arraybuffer` import would inline the bytes but blow up the worker bundle size — better to keep the font as a separately-cacheable asset.
  - GitHub blob URLs (github.com/.../raw/...) redirect to an HTML page when the path doesn't exist; curl follows redirects silently so you end up with a fake TTF that is actually HTML. `file` on the download is the one-second check — always run it before using a downloaded font. In this story I burned two tries (one on blob, one on raw.githubusercontent 404) before pulling the official release zip.
  - Worker bundle grew from ~611kb (server index) + 1.5MB (router) to +2.5MB (resvg wasm) + ~600KB (fonts) + yoga wasm. CF Workers limits are 10MB compressed, so we're fine, but any further WASM additions (e.g. image resizing) need size budgeting.
  - US-011 (wiring og:image per route) should cap query-string length to keep unfurlers happy; Twitter caches per URL so long titles will create N distinct cache keys. The 80-char cap in `parseParams` pre-empts that.
